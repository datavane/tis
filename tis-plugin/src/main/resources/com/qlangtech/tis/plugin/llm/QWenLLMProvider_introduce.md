# 通义千问大模型插件简介

## 插件简介

通义千问（QWen）大模型插件是TIS平台与阿里云通义千问AI服务的连接桥梁。通义千问是阿里云推出的企业级大语言模型服务，具备强大的自然语言理解和生成能力。本插件让TIS用户无需编写代码即可将通义千问的AI能力集成到数据处理流程中，实现智能化的数据分析、内容生成和知识问答等功能。

通过简单的配置，您就可以让TIS数据管道具备AI推理能力，从传统的数据搬运工具升级为智能数据处理平台。

## 核心作用

1. **AI能力接入**：提供与阿里云通义千问大模型的标准化对接接口，屏蔽底层API调用复杂性
2. **多模型支持**：支持通义千问全系列模型（Turbo快速版、Plus均衡版、Max高质量版、Coder代码专家版、Max长文本版），可根据不同场景选择最合适的模型
3. **结构化输出**：支持JSON Schema约束的结构化数据输出，确保AI返回的内容符合预定义的数据格式，便于后续数据处理
4. **智能重试机制**：内置请求失败自动重试功能，提升服务稳定性，减少因网络波动导致的任务中断
5. **Token使用统计**：实时追踪模型调用的Token消耗情况，帮助控制成本和优化提示词设计

## 适用场景

1. **智能数据标注**：利用大模型自动为数据打标签、分类或提取关键信息，减少人工标注成本
2. **内容智能生成**：基于数据源内容自动生成摘要、描述、文案等衍生内容，丰富数据维度
3. **数据质量检查**：通过AI判断数据的合理性、完整性，识别异常数据或数据质量问题
4. **知识抽取增强**：从非结构化文本中提取结构化信息（如实体、关系、属性），构建知识图谱
5. **ChatBI智能问答**：为数据分析场景提供自然语言交互能力，让用户用日常语言查询数据

## 工作机制

插件通过HTTP协议与阿里云通义千问API服务通信。当数据处理流程触发AI推理需求时，插件会将用户提示词（Prompt）和系统指令发送到指定的通义千问模型，模型处理后返回生成的文本内容或结构化数据。

插件在配置保存时会自动执行连通性测试，向通义千问API发送测试请求，确保API密钥有效且服务可达。只有测试通过后配置才能生效，从而避免因配置错误导致的运行时故障。

对于需要结构化输出的场景，插件会在请求中附加JSON Schema定义，并在提示词中强化格式要求，引导模型严格按照Schema规范输出数据。返回结果经过JSON解析验证，确保数据格式符合预期。

## 使用建议

1. **模型选择策略**：对于代码相关任务（如SQL生成、代码分析）推荐使用qwen3-coder-plus；一般任务使用qwen-plus即可满足需求；对需要处理长文本（如长文档分析）的场景选择qwen-max-longcontext
2. **参数调优**：temperature参数控制输出的随机性，建议从0.1开始调整；需要创意性内容时可适当提高（0.7-1.0），需要确定性结果时保持较低值（0.1-0.3）
3. **成本控制**：合理设置maxTokens参数限制单次调用的最大输出长度，避免不必要的Token消耗；在开发调试阶段建议开启"打印日志"便于问题排查，生产环境关闭以减少日志量
4. **安全管理**：API密钥具有账户权限，请妥善保管，不要泄露；建议为TIS平台单独申请专用API密钥，便于权限隔离和使用追踪
